# AI 에이전트 협업 가이드

AI 에이전트(코덱스/챗봇)에게 작업을 맡길 때, 서로 수정 범위를 침범하지 않도록 하기 위한 규칙 문서입니다.

## 1) 사용 방법

에이전트 요청 프롬프트에 아래 문장을 항상 포함합니다.

```text
반드시 AI_에이전트_협업_가이드.md를 먼저 읽고 준수해.
지정한 수정 허용 범위 밖 파일은 절대 수정하지 마.
```

## 2) 핵심 원칙

1. 지정된 파일/폴더 외 수정 금지
2. 기존 변경사항 덮어쓰기 금지
3. 이유 없는 대규모 포맷팅/리네임 금지
4. 빌드 산출물 수정은 명시 요청 시에만 허용
5. 시작 전/종료 후 반드시 변경 파일 목록 보고

## 3) 작업 범위 선언 규칙

에이전트는 작업 전에 아래 형식으로 범위를 선언해야 합니다.

```text
[작업 범위 선언]
- 수정 허용: <파일 또는 폴더>
- 수정 금지: <수정 허용 외 전체>
- 생성 허용: <필요 시 파일>
- 검증 방법: <build/test/lint 등>
```

## 4) 파일 소유권 기본 규칙

별도 지시가 없으면 아래 소유권을 따릅니다.

- 프론트 UI: `src/**`
- 백엔드 API: `backend/src/main/java/**`
- 배포/인프라: `backend/Dockerfile`, `backend/build.gradle`, `.dockerignore`
- 문서: `README.md`, `브랜치_전략_가이드.md`, `AI_에이전트_협업_가이드.md`
- 빌드 산출물: `backend/src/main/resources/static/**` (요청 시만 수정)

## 5) 작업 잠금(동시 수정 방지)

같은 파일을 동시에 수정하지 않기 위해 아래 규칙을 사용합니다.

1. 작업 시작 시 “잠금 파일 목록”을 선언
2. 다른 작업자가 잠근 파일은 수정 금지
3. 작업 완료 후 잠금 해제 선언

권장 선언 예시:

```text
[잠금]
src/pages/HomePage.tsx
src/layouts/AppLayout.tsx
```

## 6) 금지 사항

아래는 사용자 명시 지시 없이는 금지합니다.

- `git reset --hard`, 강제 되돌리기
- 인증/보안 설정 임의 수정 (`SecurityConfig`, `CorsConfig`, JWT)
- 대규모 자동 포맷/정렬 커밋
- 배포 파일과 기능 파일을 한 커밋에 혼합

## 7) 완료 보고 템플릿

작업 종료 시 아래 형식으로 보고합니다.

```text
[완료 보고]
- 수정 파일:
- 주요 변경:
- 범위 외 수정 여부: 없음/있음
- 검증 결과:
- 남은 리스크:
```

## 8) PR/커밋 규칙

- 커밋 접두사: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`
- 한 커밋에는 한 목적만 포함
- 범위 외 파일이 섞이면 커밋 중단 후 재정리

## 9) 요청 프롬프트 템플릿

아래 템플릿을 복붙해서 사용합니다.

```text
반드시 AI_에이전트_협업_가이드.md를 먼저 읽고 준수해.

[작업]
<요청 내용>

[수정 허용 범위]
<파일/폴더>

[수정 금지 범위]
<파일/폴더 또는 "허용 범위 외 전체">

[검증]
가능하면 build/test를 실행하고 결과까지 보고해.
```

## 10) 예시

```text
반드시 AI_에이전트_협업_가이드.md를 먼저 읽고 준수해.

[작업]
홈 화면 모바일 캘린더 하단 잘림 수정

[수정 허용 범위]
src/pages/HomePage.tsx
src/layouts/AppLayout.tsx

[수정 금지 범위]
backend/**, Dockerfile, README.md

[검증]
npm run build 실행 후 결과와 수정 파일 목록 보고
```
